Wireless Communication

ESP NOW

For wireless communication, we can use ESP-NOW protocol which is a connectionless protocol that allows devices to communicate without the need for a traditional Wi-Fi network. This is particularly useful for low-power devices that need to send small amounts of data quickly.

ESP-NOW is supported by ESP32 and ESP8266 micro controllers, which are popular choices for IoT applications. The protocol allows for direct communication between devices, enabling fast and efficient data transfer without the overhead of establishing a full Wi-Fi connection.

WiFi (With Optimization Strategies)

To ensure reliable communication, we can implement WiFi optimization strategies such as using WiFiMulti for redundancy. This allows the device to connect to multiple Wi-Fi networks and automatically switch between them if one becomes unavailable, ensuring continuous connectivity.

Additionally, we can analyze network performance to identify potential bottlenecks and optimize throughput. This may involve adjusting Wi-Fi settings, such as channel selection and bandwidth, to improve overall performance.

Security is also a critical consideration. We can implement data encryption and secure transmission protocols to protect sensitive information during transmission. This is especially important when dealing with personal health data, such as prescriptions.

With ESP-NOW we still need to send the data to cloud. We can chose to do the processing offline and only send purely useful data. But in WIFI we have to send everything to a local server (at the clint location) or main server.

With WIFI / cellular the system is purely portable but with ESP-NOW it need an intermediate device.

Micro controller

For the micro controller, we can use ESP32 or ESP8266. Both of these micro controllers are capable of handling the required tasks and have built-in support for Wi-Fi and Bluetooth communication.
